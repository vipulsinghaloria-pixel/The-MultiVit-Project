# Project Description

Choosing the right multivitamin can be confusing due to the wide variety of brands available and the lack of structured comparison. This project aims to solve this problem by building a SQL-based analysis system to evaluate and compare multivitamin brands.

The project uses three datasets:
1. Nutritional composition data (extracted using AI from product labels).
2. Market data including ratings and purchase volumes from Amazon and Flipkart.
3. Created a third table (multivit3), containing standardized Recommended Daily Allowance (RDA) values to represent the required daily intake of each nutrient for humans.

# Using PostgreSQL, the data is organized into structured tables and analyzed using SQL queries to:
- Compare nutrient values across brands
- Identify strengths and deficiencies in each product
- Analyze customer ratings and demand
- Build a scoring model to recommend the best multivitamin

# Data Preprocessing & Visualization- EXCEL( Pivot Table), Pixel

- Performed data cleaning and preprocessing using Microsoft Excel, including removal of non-relevant attributes, handling inconsistencies, and standardizing units across datasets.
- Ensured data integrity and schema readiness before ingestion into PostgreSQL.
- Designed and developed visual assets for dashboard representation using Pixel, enhancing data storytelling and user interpretability.




This project showcases practical skills in data collection, data cleaning, database design, SQL querying, and real-world problem solving.
